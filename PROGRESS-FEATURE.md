## Open

- [ ] **Copy/paste annotation elements** — duplicate any overlay element (arrow, text, sticker, blur rect, pin) via `⌘C` / `⌘V` on the editor stage. Preserve type + style; offset paste position so it's visible.
- [ ] **Auto-shrink large captures** — cap exported image at a configurable max dimension (default e.g. 2560px long edge) to keep file sizes sane on 5K/6K displays. Setting lives in Output tab; "Original" option to disable.
- [ ] **Layout-independent global shortcuts** — bind hotkeys by physical key (USB HID / ANSI/ISO position) rather than the typed character, so `⌘⇧P` still fires when the active input source is Thai (where the same physical key produces `ญ`), Russian, Dvorak, etc. `tauri-plugin-global-shortcut` resolves accelerators by Unicode char on most paths, so a US-layout binding silently breaks when the user switches IME. Investigate: macOS `kVK_*` keycodes via `CGEventSourceKeyboardType` / `UCKeyTranslate` round-trip, Windows `MapVirtualKey(MAPVK_VSC_TO_VK)` with scancodes. Likely needs a custom shortcut layer below the plugin, or upstream patch. Settings UI should display the layout-localized label for the bound physical key.

## Landed

- [x] **Editor zoom + pan (landed 2026-05-26)** — cursor-anchored zoom via Ctrl/Cmd+wheel on Konva stage `scaleX/scaleY` ([src/components/editor/EditorStage.tsx](src/components/editor/EditorStage.tsx) + new [src/lib/zoom.ts](src/lib/zoom.ts) `clampZoom` + [src/lib/stageBridge.ts](src/lib/stageBridge.ts) scroll-container handle). Plain wheel scrolls vertical, Shift+wheel horizontal, middle-mouse drag pans (cursor flips to `grabbing` on stage container + body, user-select disabled — fix b195ad6 in [EditorStage.tsx](src/components/editor/EditorStage.tsx)). Viewport-sized empty margin around image (Figma-style) so any pixel reachable at any zoom; fit uses 32px inset. Toolbar gains zoom in/out, fit-to-window, 100%-reset, percentage label (click = reset) — [src/components/editor/Toolbar.tsx](src/components/editor/Toolbar.tsx). Keyboard shortcuts `⌘+ / ⌘- / ⌘0 / ⌘1` in [src/hooks/useEditorShortcuts.ts](src/hooks/useEditorShortcuts.ts). Zoom state per-editor-window in-memory via [src/stores/editor.ts](src/stores/editor.ts) (`displayScale` + `zoomFit`, excluded from undo/redo); image refits + recenters on new capture, clipboard paste, `editor:clear` via image-identity tracking in `useImage`. **Native-resolution export preserved**: [src/lib/exportImage.ts](src/lib/exportImage.ts) uses explicit-region `toDataURL` keyed on published image size so zoom/pan don't affect output pixels. Verified `pnpm tsc --noEmit` clean; user-confirmed on macOS dev build (commits 88bfc8e + b195ad6, shipped v0.2.1).

- [x] **In-editor capture buttons (full / area / window) (landed 2026-05-26)** — three icon buttons (Monitor/Crop/AppWindow) at the start of the editor toolbar invoke a new `trigger_capture_command(kind)` Tauri command. Command hides the editor window (new `windows::hide_editor_and_wait` mirrors the existing overlay-hide helper, with DwmFlush settle on Windows) and then dispatches the overlay on the main thread — running off-main caused the macOS crash hit during initial testing. Hotkey path (`shortcuts::emit_trigger`) now routes through the same `trigger_capture` so global shortcuts fired while the editor is focused also hide it before capture. Re-show paths: success via existing `load_editor_image` → `show_editor`; user-cancel via `commands::overlay::close_overlay_command` → new `show_editor_if_hidden`. Tooltips read the live-bound accelerator from `config.hotkeys` via existing `formatShortcut`. Files: [src-tauri/src/windows.rs](src-tauri/src/windows.rs), [src-tauri/src/capture_dispatch.rs](src-tauri/src/capture_dispatch.rs), [src-tauri/src/commands/capture.rs](src-tauri/src/commands/capture.rs), [src-tauri/src/commands/overlay.rs](src-tauri/src/commands/overlay.rs), [src-tauri/src/shortcuts.rs](src-tauri/src/shortcuts.rs), [src-tauri/src/lib.rs](src-tauri/src/lib.rs), [src/components/editor/Toolbar.tsx](src/components/editor/Toolbar.tsx). - [x] **Area capture — "remember last region" toggle (landed 2026-05-26)** — Settings → General toggle `Remember last area-capture region`. When on, opening the area overlay on the same monitor prefills the prior selection rect (frozen — mousemove ignored while prefilled so it can't be destroyed by accident). Enter confirms, Esc cancels, mousedown discards the prefill and starts a fresh drag. Region persisted to `config.lastUsed.region = { monitorId, x, y, w, h }` via `tauri-plugin-store` after each successful capture. Overlay capability gained `store:default`. Did *not* skip the overlay entirely (the "instant recapture without UI" variant in original spec) — felt error-prone given monitor geometry changes; current flow keeps the overlay as a visual confirmation step. Files: [src/lib/config.ts](src/lib/config.ts), [src/stores/settings.ts](src/stores/settings.ts), [src/components/settings/SettingsView.tsx](src/components/settings/SettingsView.tsx), [src/app/overlay/page.tsx](src/app/overlay/page.tsx), [src-tauri/capabilities/overlay.json](src-tauri/capabilities/overlay.json). Verified `pnpm tsc --noEmit` clean.
