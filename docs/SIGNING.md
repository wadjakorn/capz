# macOS Signing + Notarization Runbook

Future-improvement doc. **Not yet executed.** Current builds ship unsigned (`signingIdentity: "-"` in [src-tauri/tauri.conf.json](../src-tauri/tauri.conf.json)) and rely on a Homebrew Cask `postflight xattr -dr com.apple.quarantine` workaround in [packaging/homebrew-cask/capz.rb](../packaging/homebrew-cask/capz.rb).

Goal of this doc: when ready, follow these steps end-to-end so the `.dmg` opens on any Mac with **no Gatekeeper prompt, no `sudo xattr`, no cask hack**.

Cross-ref: [PLAN.md §15](../PLAN.md) (Phase 15) — high-level pass. This file is the concrete runbook.

---

## Path chosen: Developer ID + Notarization (not Mac App Store)

| | Dev ID + Notarize ✅ | Mac App Store ❌ |
|---|---|---|
| Distribution | Direct (GitHub Releases, Homebrew Cask, DMG download) | Apple App Store only |
| Tauri auto-updater | Works as-is | **Forbidden** — Apple ships updates |
| Global hotkeys via `tauri-plugin-global-shortcut` | Works | Sandbox/NSEvent global monitor flagged in review |
| Screen Recording (TCC) | Standard runtime prompt | Extra entitlement review |
| Cost | $99/yr Apple Developer Program | $99/yr + sandbox rework |
| Existing GH-pages `latest.json` endpoint | Stays live | Becomes dead weight |

MAS path is incompatible with both the screenshot tool's required APIs and the existing updater channel at `https://wadjakorn.github.io/capz/latest.json`. Dev ID + Notarize is the only sensible target.

## Step 1 — Apple Developer enrollment (off-repo, blocks everything else)

1. Enroll at https://developer.apple.com/programs/ — **$99/yr**.
   - **Individual** is fastest (~24–48h approval).
   - **Organization** needs a D-U-N-S number (~1 extra week). Only pick this if a legal entity owns the app.
2. Record your **Team ID** (10-char) from https://developer.apple.com/account → Membership.
3. Create an **app-specific password** at https://appleid.apple.com → Sign-In and Security → App-Specific Passwords. Label it `capz-notary`.
   - This is for `notarytool`. **Not** your Apple ID password.

---

## Step 2 — Create the Developer ID Application certificate

Two ways — pick one.

### A. Via Xcode (easiest)
Xcode → Settings → Accounts → add Apple ID → Manage Certificates → `+` → **Developer ID Application**.

### B. Via Apple Developer portal
https://developer.apple.com/account/resources/certificates → `+` → **Developer ID Application** → upload a CSR generated by Keychain Access → Certificate Assistant → Request a Certificate From a Certificate Authority (Saved to disk).

### Verify and export

```bash
# Should list: "Developer ID Application: <Your Name> (<TEAMID>)"
security find-identity -v -p codesigning
```

Export `.p12` for CI:
1. Keychain Access → My Certificates → right-click `Developer ID Application: ...` → Export → save `.p12` with a strong password.
2. Base64 for GitHub Actions:
   ```bash
   base64 -i DeveloperID.p12 -o DeveloperID.p12.b64
   ```

## Step 3 — Patch `src-tauri/tauri.conf.json`

Replace the `bundle.macOS` block:

```json
"macOS": {
  "signingIdentity": "Developer ID Application: <Your Name> (<TEAMID>)",
  "providerShortName": "<TEAMID>",
  "entitlements": "entitlements.plist",
  "hardenedRuntime": true,
  "minimumSystemVersion": "12.0"
}
```

- **Hardened Runtime is required for notarization.** Apple rejects non-hardened submissions.
- `signingIdentity` in JSON keeps local `pnpm tauri build` working. CI overrides via `APPLE_SIGNING_IDENTITY` env var (Step 5).

---

## Step 4 — Create `src-tauri/entitlements.plist`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>com.apple.security.cs.allow-jit</key><true/>
  <key>com.apple.security.cs.allow-unsigned-executable-memory</key><true/>
  <key>com.apple.security.cs.disable-library-validation</key><true/>
  <key>com.apple.security.automation.apple-events</key><true/>
</dict>
</plist>
```

Why each:
- `allow-jit` + `allow-unsigned-executable-memory` — WKWebView JS engine needs JIT.
- `disable-library-validation` — Tauri loads dylibs (xcap deps, plugins) not signed by your team.
- `apple-events` — safe to include; useful if anything ever shells to AppleScript.

**No `com.apple.security.app-sandbox` key** → app is not sandboxed. Screen Recording grant flows through TCC (existing onboarding handles it). Global shortcuts work without extra entitlement.

## Step 5 — GitHub Actions secrets

Repo → Settings → Secrets and variables → Actions → New repository secret.

| Secret name | Value |
|---|---|
| `APPLE_CERTIFICATE` | Contents of `DeveloperID.p12.b64` (the base64 file) |
| `APPLE_CERTIFICATE_PASSWORD` | The `.p12` export password from Step 2 |
| `APPLE_SIGNING_IDENTITY` | `Developer ID Application: <Your Name> (<TEAMID>)` |
| `APPLE_ID` | Apple ID email |
| `APPLE_PASSWORD` | App-specific password from Step 1 (the `capz-notary` one) |
| `APPLE_TEAM_ID` | 10-char Team ID |
| `KEYCHAIN_PASSWORD` | Any random throwaway string — CI creates a temp keychain with it |

**Additive** to existing `TAURI_SIGNING_PRIVATE_KEY` / `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` (Tauri updater key — see Pitfalls). Total macOS-related secrets after this step: ~9.

## Step 6 — Update `.github/workflows/build.yml`

`tauri-apps/tauri-action@v0` already understands these env vars natively — no shell scripting needed. Add to the existing `Build + release via tauri-action` step's `env:` block:

```yaml
- name: Build + release via tauri-action
  uses: tauri-apps/tauri-action@v0
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
    TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
    TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
    # NEW — macOS code signing + notarization
    APPLE_CERTIFICATE: ${{ secrets.APPLE_CERTIFICATE }}
    APPLE_CERTIFICATE_PASSWORD: ${{ secrets.APPLE_CERTIFICATE_PASSWORD }}
    APPLE_SIGNING_IDENTITY: ${{ secrets.APPLE_SIGNING_IDENTITY }}
    APPLE_ID: ${{ secrets.APPLE_ID }}
    APPLE_PASSWORD: ${{ secrets.APPLE_PASSWORD }}
    APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
    KEYCHAIN_PASSWORD: ${{ secrets.KEYCHAIN_PASSWORD }}
  with:
    # ... existing args unchanged ...
```

What tauri-action does with these:
1. Imports `.p12` into a temp keychain.
2. Signs `.app` with Hardened Runtime + entitlements.
3. Builds `.dmg`.
4. Submits to `notarytool`, waits for ticket.
5. Staples the ticket to the `.dmg` and the `.app` inside `.app.tar.gz`.

Windows matrix entry is untouched — Windows code signing is a separate problem (see "Out of scope").

## Step 7 — Remove the Homebrew Cask quarantine bypass

After the first signed/notarized release ships and is verified, edit [packaging/homebrew-cask/capz.rb](../packaging/homebrew-cask/capz.rb):

- Delete the entire `postflight do ... end` block (the `xattr -dr com.apple.quarantine` call).
- Delete the trailing comment about removing it once signed.
- Leave `auto_updates false`, `depends_on macos: ">= :ventura"`, `app "capz.app"` alone.

After this, `brew install --cask capz` installs a properly signed `.app` and Gatekeeper passes silently.

## Verification

### Local sanity (before pushing a tag)

```bash
# 1. Build with the new config + your cert in login keychain
pnpm tauri build --target aarch64-apple-darwin

APP=src-tauri/target/aarch64-apple-darwin/release/bundle/macos/capz.app

# 2. Signature
codesign -dv --verbose=4 "$APP"
# Expect:
#   Authority=Developer ID Application: <Name> (<TEAMID>)
#   flags=0x10000(runtime)   ← hardened runtime enabled

# 3. Entitlements
codesign -d --entitlements - "$APP"

# 4. Pre-notarize Gatekeeper check
spctl -a -t exec -vv "$APP"
# Will say "rejected" until stapled — that's expected before notarization.
```

### End-to-end (CI tag push)

1. Push a tag:
   ```bash
   git tag v0.1.9 && git push --tags
   ```
2. Watch GitHub Actions — the tauri-action step should log notarization submission + acceptance (~5–15 min).
3. Download the `.dmg` from the draft release on a **fresh Mac** (or one that has never run capz before — quarantine attribute is per-file):
   ```bash
   stapler validate capz_0.1.9_aarch64.dmg
   # → "The validate action worked!"

   spctl -a -t open --context context:primary-signature -vv capz_0.1.9_aarch64.dmg
   # → "source=Notarized Developer ID"
   ```
4. Double-click `.dmg` → drag to Applications → launch. **Zero Gatekeeper prompt.** (First-launch TCC prompts for Screen Recording / Accessibility are normal — those are runtime user grants, unrelated to signing.)
5. Auto-update sanity: install previous unsigned/signed version, point updater at the new `latest.json`, confirm in-app update path still works. Tauri updater signature is orthogonal to Apple cert signature.

## Critical pitfalls

- **Two distinct signing layers, never conflate them:**
  - **Tauri updater Ed25519 key** (`TAURI_SIGNING_PRIVATE_KEY`) — signs `.app.tar.gz` bundles for in-app updates. Loss = every existing user stranded on their current version forever.
  - **Apple Developer ID cert** — signs the `.app` binary for Gatekeeper. Loss = create a new one ($0, Apple portal); existing notarized binaries keep working.
  - Both are required. Neither replaces the other.
- **Notarization takes 5–15 min over network.** If CI times out, check https://developer.apple.com/system-status/. Inspect failures via:
  ```bash
  xcrun notarytool log <submission-id> --apple-id <APPLE_ID> --team-id <APPLE_TEAM_ID> --password <APPLE_PASSWORD>
  ```
- **Hardened Runtime + dylib loading**: if the signed app crashes on launch with `Library Validation failed` in Console.app, you're missing `com.apple.security.cs.disable-library-validation`. Double-check `entitlements.plist`.
- **Cert expiry**: Developer ID Application certs last **5 years**. Notarized binaries keep working past cert expiry (Gatekeeper checks the stapled ticket, not cert validity). But you can't sign **new** builds after expiry — renew via developer portal.
- **First sign of trouble**, before re-tagging, run `xcrun notarytool history --apple-id ... --team-id ... --password ...` to see queue state.

## Out of scope

- **Windows code signing** — separate cert chain (Sectigo / DigiCert EV ~$200–400/yr, or Azure Trusted Signing). Different env vars, different keychain. Tackle as its own runbook.
- **Mac App Store submission** — see comparison table above; deliberately rejected.
- **Provisioning profiles** — only needed for MAS or capabilities like Push Notifications, iCloud, CloudKit, Apple Pay. Dev ID direct distribution does not use them.
- **Sparkle** — Tauri's built-in updater already covers our needs; no migration.
