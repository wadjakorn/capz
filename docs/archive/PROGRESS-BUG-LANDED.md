- [x] **macOS update invalidates Screen Recording grant — detection + UX (landed 2026-05-25)**. Confirmed root cause: [src-tauri/tauri.conf.json](../../src-tauri/tauri.conf.json) ships `"signingIdentity": "-"` (ad-hoc) so every build's cdhash changes; TCC keys the grant by code identity, so the System Settings row keeps the prior cdhash and the new binary reads denied. Real fix is Developer ID Application signing — left as a follow-up in Open. This change ships the detection-and-prompt layer:
  - New [useStalePermissionAfterUpdateListener](../../src/lib/notice.ts) hook (macOS-only). On mount: reads `has_screen_recording_permission`, the current binary version via `@tauri-apps/api/app#getVersion`, and a `permissions.lastGrantedVersion` marker in [config.json](../../src/lib/config.ts) via `@tauri-apps/plugin-store`.
  - If granted now and the marker doesn't match the current version → writes current version as the marker. First install with no marker stays silent (onboarding owns the never-granted path).
  - If denied now and a marker exists for a *different* prior version → surfaces a 20s `toast.error` with stable id `permission-stale-after-update`, copy explaining macOS keeps the prior entry, and an "Open Privacy Settings" action wired to `open_system_settings_screen_recording`.
  - Wired in [src/app/editor/page.tsx](../../src/app/editor/page.tsx) next to the existing `usePermissionRevokedListener`. Verified clean — `pnpm tsc --noEmit`.

- [x] **Dark overlay baked into captured image (landed 2026-05-26)** — root cause: Windows DWM "Animate windows when minimizing/maximizing" transition. `win.hide()` on overlay HWND triggers DWM shrink animation; xcap (BitBlt) reads the desktop DC mid-animation and bakes a shrunken overlay frame into the capture. Per-machine repro tracked the OS Performance Options setting, not the iGPU/dGPU split (AMD Radeon 680M vs RTX 4050 was correlation, not causation). Earlier `DwmFlush()` fix (commit c6ee387) was insufficient — the next composite is still mid-animation, so flush waits one frame but doesn't kill the transition.
  - **Primary fix**: `DwmSetWindowAttribute(hwnd, DWMWA_TRANSITIONS_FORCEDISABLED, TRUE)` per overlay HWND right before `win.show()` in [windows::show_overlay_mode](../../src-tauri/src/windows.rs). `hide()` becomes instant; `DwmFlush()` backstop now deterministic.
  - **Defense in depth**: new `set_overlay_cutout` command + `apply_overlay_region` helper carves a real HWND hole at the selection rect via `CreateRectRgn` + `CombineRgn(RGN_DIFF)` + `SetWindowRgn`. Frontend RAF-coalesces calls during area drag / window hover. Even if DWM ever held a stale frame, the inner region is genuinely outside the overlay's composition.
  - **UI redesign**: [src/app/overlay/page.tsx](../../src/app/overlay/page.tsx) replaces the uniform `rgba(0,0,0,0.35)` backdrop with 4-div outer-dim around the selection (transparent inner). Cross-platform visual via CSS; SetWindowRgn for Windows compositor truth.
  - Added `windows-sys` Win32_Foundation + Win32_Graphics_Gdi + Win32_UI_WindowsAndMessaging features. Reverted experimental `close()`-instead-of-`hide()` variant in `hide_overlays_and_wait`.
  - Verified `pnpm tsc --noEmit` + `cargo clippy --all-targets -- -D warnings` (macOS target). Windows code paths only syntactically verified — pending interactive verification on the bad Windows machine.

- [x] **Windows captures look low-quality regardless of format/quality settings** — root cause was **in our own code**, not xcap/BitBlt/DPI. Two stacked losses in [src-tauri/src/services/image_service.rs](../../src-tauri/src/services/image_service.rs): (1) unconditional JPEG 85 re-encode of every intermediate buffer (hits 1080p Windows captures too — confirmed by user report, where output set to PNG didn't help because Konva was already loading from the lossy JPEG temp file), (2) unconditional Triangle downscale to 2560 longest edge (stacked on top for 4K/5K/6K monitors). Fix landed: split encoder into `write_temp_png` (lossless, full-res default) + `write_temp_jpeg(quality, max_edge)` (opt-in); Rust now dispatches via new `IntermediateSpec` read from store keys `capture.intermediateFormat` + `capture.intermediateMaxEdge`; Settings → Capture intermediate exposes both knobs. Default post-fix: PNG, no edge cap. Final PNG export from editor is now pixel-perfect vs source. **E2E verification pending** — Windows machine cannot run `pnpm tauri dev`; needs validation against a packaged build (.msi) on the affected 1080p Windows host before closing.

- [x] **macOS post-update permission grant becomes inert (toggle-off doesn't help; full remove + relaunch + re-onboard required)** — distinct from the existing stale-TCC-after-update detection: after an update, System Settings → Screen Recording still shows capz checked + green, but every capture attempt fails. Toggling the row off then on does NOT recover. The only working workaround: **remove** the capz entry from the list entirely (− button), relaunch the app, walk through onboarding so TCC re-prompts and writes a fresh entry. Likely same root cause as the existing item (ad-hoc `signingIdentity: "-"` → cdhash changes per build → TCC entry now points at a stale code identity, but the OS won't re-prompt because the bundle ID still matches). Real fix is Developer ID signing; interim: detect this state and surface a one-click guide that opens System Settings + walks the user through Remove → Relaunch → Re-grant. Existing `useStalePermissionAfterUpdateListener` only covers the *denied* case — this one shows *granted* in the UI but capture still fails, so detection has to be capture-probe based (see "Onboarding 'Granted' sticks after mid-session revoke").

Registered in [src-tauri/src/lib.rs](../../src-tauri/src/lib.rs). New `useInertGrantAfterUpdateListener` ([src/lib/notice.ts](../../src/lib/notice.ts)) — macOS-only, runs on mount + window focus, skips when preflight denied (the stale-hook owns that path) or when `permissions.lastProbeOkVersion === currentVersion`. Settings → General gains macOS-only "Fix permission after macOS update" row as a non-toast entry point ([src/components/settings/SettingsView.tsx](../../src/components/settings/SettingsView.tsx)). Wired in [src/app/editor/page.tsx](../../src/app/editor/page.tsx). `cargo clippy --all-targets -- -D warnings` + `pnpm tsc --noEmit` clean.
